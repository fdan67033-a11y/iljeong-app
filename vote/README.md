# 청렴 슬로건 공모·투표 시스템

지방세 고지서·모바일 알림톡에 실을 청렴 슬로건을 부서원이 직접 제안하고 투표로
선정하는 시스템입니다. 슬로건 접수(7월~) → 투표(8월 말, 관리자가 개시) → 마감의
3단계로 운영되며, 저장·검증은 Firebase Realtime Database가 담당합니다.

## 페이지 구성

| 페이지 | 용도 |
|---|---|
| `index.html` | 참가자 페이지 (개인 QR 전용). 접수 단계엔 문구 제출+갤러리, 투표 단계엔 1인 2표 투표 |
| `board.html` | 현황판. 접수 중엔 슬로건 갤러리, 투표 중엔 실시간 득표 순위, 마감 후엔 👑 선정 문구 |
| `admin.html` | 관리자 페이지. 구글 로그인 후 단계 전환(접수→투표→마감) 버튼 + 현황 |
| `config.js` | 설정: 팀, 명단(코드의 해시), Firebase 접속 주소 |

관리자 PC에만 보관하는 파일 (저장소에 없음 — 공유 금지):
`vote-qr-cards.html`(개인 QR 카드 인쇄 + 관리자 주소), `firebase-seed.json`(명단+비밀값),
`firebase-rules.json`(보안 규칙 원본)

## 운영 규칙 — 전부 서버(Firebase 보안 규칙)가 강제

- 문구 제출은 접수(collect) 단계에만, 투표는 투표(vote) 단계에만 가능
- 제출된 문구는 자동으로 제출자의 팀 소속이 됨 (개인 QR로 인증, 화면에는 팀만 표시)
- 1인 2표, 서로 다른 문구 2개, **자기 팀 문구 투표 불가**
- 위조 불가: 제출·투표 모두 개인 코드가 서버 명단과 일치해야만 저장
- 1회 제출 확정: 투표는 수정 불가. 착오 시 관리자가 콘솔에서 그 사람 표만 삭제하면 재투표 가능

## 진행 순서

1. `vote-qr-cards.html` 인쇄 → 개인 QR 카드 배부
2. 접수 기간: 각자 QR로 접속해 문구 제출 (팀별 1개 이상), 갤러리에서 서로 열람
3. 투표 개시: 관리자가 admin.html에 로그인해 "투표 개시" 클릭 → 모든 화면이 즉시 투표 모드로
4. 투표: 자기 팀 제외 2개 선택 → 제출 (확정)
5. 마감: 관리자가 admin.html에서 "투표 마감" 클릭 → 현황판에 최다득표 문구 👑 확정 표시

## 관리 (Firebase 콘솔 → Realtime Database → 데이터 탭)

단계 전환·삭제 모두 콘솔에서 하며, 콘솔은 관리자 구글 계정 로그인이 필요하므로
공모 진행 권한은 관리자만 갖습니다 (페이지 주소가 알려져도 조작 불가).

- 단계 전환: admin.html 로그인 후 단계 버튼 (또는 콘솔에서 state/phase 직접 편집)
- 관리자 계정 설정: FIREBASE.md의 "관리자 계정 설정" 참고 (Authentication에 계정 1개 생성)
- 부적절한 문구 삭제: `polls/slogan-2026/sloganList/<항목>` 삭제
- 한 사람 재투표 허용: `polls/slogan-2026/ballots/<그 사람 해시>` 삭제
- 새 공모 시작: config.js의 id 변경 + 시드 JSON을 새 경로에 가져오기

## 이름/팀 바꾸기

세 곳을 같은 순서로 수정: ① config.js의 `voters`(name/team), ② QR 카드 파일 명단,
③ Firebase의 `voters/<해시>/public`. `h`(해시)와 QR 카드의 개인 코드는 짝이므로 유지.
팀 이름(`teams`)은 config.js와 시드의 팀 key가 일치해야 합니다.
