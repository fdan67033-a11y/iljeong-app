# QR 실시간 투표 (데모)

행사장에서 결과 화면을 띄워두고, 참가자가 QR을 찍어 투표하면 그래프가 실시간으로 움직이는 데모입니다.

## 사용법

| 페이지 | 주소 | 용도 |
|---|---|---|
| 결과 화면 | `board.html` | 빔프로젝터/대형 화면에 띄우는 페이지. QR 코드 + 실시간 그래프 |
| 투표 페이지 | `index.html` (폴더 주소로 접속) | 참가자가 QR을 찍으면 열리는 페이지 |

1. 발표용 PC에서 `board.html`을 엽니다.
2. 참가자가 화면의 QR을 스마트폰 카메라로 찍고 투표합니다.
3. 그래프가 1초 안에 자동으로 움직입니다. 새로고침 불필요.

## 질문/선택지 바꾸기

`config.js` 하나만 수정하면 됩니다. 새 투표를 시작하려면 `id`의 `v1`을 `v2`로 바꾸세요.

## 데모의 한계 (실전 전환 시 알아둘 것)

지금은 가입이 필요 없는 공개 MQTT 브로커(broker.emqx.io)로 실시간을 구현했습니다.
빠르게 눈으로 확인하기엔 충분하지만:

- 집계가 결과 화면(board.html)을 열어둔 브라우저에만 저장됩니다 (localStorage).
- 공개 브로커라 악의적인 사용자가 채널에 끼어들 수 있습니다.
- 중복 투표 방지가 기기별 localStorage 수준입니다 (테스트용 재투표 버튼 있음).

실제 행사에서 쓰려면 Firebase Realtime Database로 백엔드만 바꾸면 됩니다
(무료, 페이지 구조는 그대로). 그러면 집계가 서버에 저장되고, 보안 규칙으로 조작도 막을 수 있습니다.
