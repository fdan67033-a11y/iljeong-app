# QR 실시간 팀 투표

20명이 개인 QR로 접속해 자기 팀을 제외한 3팀에 투표하고, 결과 페이지는 누구나 언제든 실시간으로 볼 수 있는 시스템입니다.

## 페이지 구성

| 페이지 | 용도 |
|---|---|
| `board.html` | 결과 페이지. 빔프로젝터용이자, 20명 전원이 각자 폰으로 열어도 됨. 언제 열어도 전체 집계가 복원됨 |
| `index.html` | 투표 페이지. 개인 QR(`?t=개인코드`)로만 참여 가능 |
| `config.js` | 설정: 제목, 팀, 명단(코드의 해시), 1인 표 수 |

개인 QR 카드 인쇄 파일(`vote-qr-cards.html`)은 전원의 비밀 코드가 들어 있어
**이 저장소에 없습니다** — 관리자 PC에만 보관하세요.

## 규칙 (config.js에서 변경 가능)

- 1인 3표, 서로 다른 3팀을 골라야 제출 가능
- 자기 팀 투표 불가 (화면에서 잠금 + 집계에서 재검증)
- 개인 코드당 1장의 표만 유효 — 다시 제출하면 마지막 표로 교체 (수정 허용)
- 명단에 없거나 위조된 표는 무시 (표에 담긴 코드의 해시가 명단과 일치해야 인정)

## 행사 진행 순서

1. 관리자 PC의 `vote-qr-cards.html` 열어서 인쇄, 이름대로 배부
2. 빔프로젝터에 `board.html` 띄우기
3. 각자 QR 찍고 3팀 선택 → 제출
4. 결과판의 QR을 찍으면 각자 폰에서도 실시간 결과 시청 가능
5. 다시 하려면 관리자용 주소 `board.html?admin=비밀번호` 로 열어 "전체 초기화"
   (또는 config.js의 id를 v3으로 바꿔 완전히 새 채널로)

## 이름/팀 바꾸기

`config.js`의 `voters`에서 `name`/`team`을 수정하면 됩니다 (`h`는 QR 카드와 짝이므로 유지).
QR 카드 파일의 명단도 같은 순서로 함께 수정하세요.

## 동작 원리와 한계

실시간 전달과 표 저장 모두 무료 공개 MQTT 브로커(broker.emqx.io, broker.hivemq.com)의
보존 메시지(retained message)를 씁니다 — 사람별 마지막 표가 두 브로커 모두에 저장되어
결과 페이지를 언제 열어도 복원됩니다. 표에는 개인 코드가 들어 있고 결과판은 코드의
해시가 명단과 일치할 때만 집계하므로, 공개된 파일만 봐서는 남의 표를 위조할 수 없습니다.

다만 가입/서버 없이 돌아가는 구조의 한계로:

- 행사 중 채널을 도청하면 개인 코드를 주울 수 있음 (도청 이후의 위조는 가능)
- 공개 브로커가 보존 메시지를 영구 보관한다는 보장은 없음 — 행사 당일용, 장기 보관 부적합
- 표 내용이 암호화되지 않아 완전한 비밀투표는 아님

성적 반영 등 공정성이 중요한 투표라면 백엔드를 Firebase로 바꿔
서버 검증 + 접근 제어를 갖추는 것을 권장합니다 (페이지 구조는 그대로 재사용 가능).
